# 测试接线说明

## 测试条件
**所有4个通道连接到同一信号源**，所有通道测试相同频率。

## 硬件连接

### 信号发生器输出
将信号发生器的输出端（SIGNAL OUT）连接到以下4个GPIO引脚：

| 通道 | GPIO编号 | 物理引脚 | 芯片线路 |
|------|-----------|----------|----------|
| 通道1 | GPIO5 | Pin 29 | /dev/gpiochip4 line 5 |
| 通道2 | GPIO6 | Pin 31 | /dev/gpiochip4 line 6 |
| 通道3 | GPIO23 | Pin 16 | /dev/gpiochip4 line 23 |
| 通道4 | GPIO24 | Pin 18 | /dev/gpiochip4 line 24 |

### 地线连接
将信号发生器的地线（GND）连接到 Raspberry Pi 的任何 GND 引脚，例如：
- Pin 6
- Pin 9
- Pin 14
- Pin 20
- Pin 25
- Pin 30
- Pin 34
- Pin 39

## 信号设置

### 信号参数
- **波形**：方波 (Square Wave)
- **幅度**：3.3V (逻辑电平)
- **占空比**：50%
- **频率**：从外部信号发生器设置（测试程序会自动适配）

### 测试频率
根据测试阶段，从外部信号发生器设置以下频率：

#### Stage 1: 多通道测试 (multi_channel_freq_test)
- 10 kHz（基准测试）
- 50 kHz
- 75 kHz
- 100 kHz

#### Stage 2: 压力测试 (high_freq_stress_test)
- 100 kHz
- 200 kHz
- 500 kHz

#### Stage 3: 延迟测试 (latency_test)
- 500 kHz
- 750 kHz
- 1000 kHz (1 MHz)

## 测试流程

### 自动运行（推荐）
```bash
sudo ./run_high_freq_tests.sh
```
选择选项 4 运行所有测试。程序会提示当前测试的频率。

### 手动运行每个测试
```bash
# Stage 1: 多通道测试
sudo ./multi_channel_freq_test

# Stage 2: 压力测试
sudo ./high_freq_stress_test

# Stage 3: 延迟测试
sudo ./latency_test
```

## 注意事项

1. **重要**：所有4个通道必须接在同一信号源上
2. 不要将不同频率的信号接到不同通道
3. 每次测试前，请确保信号发生器已设置到正确的频率
4. 信号幅度不应超过 3.3V，以免损坏 GPIO
5. 测试过程中请勿更改信号频率
6. 建议使用屏蔽线减少干扰

## 预期结果

### Stage 1 通过标准
- 所有4个通道的边沿丢失率 < 5%
- 各通道测量的频率与设置频率误差 < 1%

### Stage 2 通过标准
- 边沿丢失率 < 10%
- CPU 使用率 < 90%
- 性能稳定（波动 < 20%）

### Stage 3 通过标准
- 触发到显示延迟 < 100 ms

## 故障排除

### 所有通道显示 0 边沿
- 检查信号发生器是否开启
- 检查接线是否正确
- 检查 GPIO 引脚是否损坏

### 某些通道边沿丢失严重
- 检查该通道的接线
- 确认所有通道都接到同一信号源
- 检查信号质量（示波器观察）

### 测试结果不稳定
- 减少系统负载（关闭其他程序）
- 检查电源供应是否稳定
- 使用更短的连接线
